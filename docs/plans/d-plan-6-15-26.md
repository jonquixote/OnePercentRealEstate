# D-Plan 6-15-26 — Foundation Upgrade

## Executive Summary

**One Percent Real Estate** has 600,000+ properties in its database but the homepage
shows only 4 passing the 1% rule — all likely vacant lots. The map shows 4 pins
across the entire US. App-generated data hasn't updated since initial setup.

This plan addresses four interconnected pillars:

| Pillar | Problem | Root Cause |
|--------|---------|------------|
| **P1** Data Pipeline | 4/~600k "1% passes" — stats stale, rents uncalculated | `rent_calc_status` backfill incomplete; non-rentable types not excluded; cache invalidation gaps |
| **P2** Property Type Rules | Vacant lots counted as rentable; no type-specific financial targets | Three independent hardcoded taxonomies (SQL, Python, ML); no `property_type_rules` table |
| **P3** Dynamic Homepage | Static "4,600+ listings" copy; stats never refresh; empty states misleading | Hardcoded numbers; single-fetch-on-mount with no polling; no distinction between "loading" and "no data" |
| **P4** Map Overhaul | 4 pins for entire US; JSON viewport API slow at scale | `mv_cluster_tiles` MV filtering gaps; pg_tileserv deployed but idle; float-zoom cache-key kills hit rate |

Three independent subagent reviews rated this plan **CONDITIONAL PASS** across
backend/data, frontend/UX, and infrastructure/ops. All flagged issues are
incorporated below.

```sql
-- Helper function: normalize a raw property_type to its canonical slug.
-- Returns 'other' if no match found.
CREATE OR REPLACE FUNCTION normalize_property_type(raw_type TEXT)
RETURNS TEXT
LANGUAGE plpgsql
IMMUTABLE
STRICT
AS $$
DECLARE
    result TEXT;
BEGIN
    IF raw_type IS NULL OR raw_type = '' THEN
        RETURN 'other';
    END IF;

    -- Exact match first
    SELECT canonical_slug INTO result
    FROM property_type_rules
    WHERE UPPER(raw_input_pattern) = UPPER(TRIM(raw_type));

    IF result IS NOT NULL THEN
        RETURN result;
    END IF;

    -- Partial match fallback: check if raw_type CONTAINS any pattern
    SELECT canonical_slug INTO result
    FROM property_type_rules
    WHERE UPPER(TRIM(raw_type)) LIKE '%' || UPPER(raw_input_pattern) || '%'
    ORDER BY priority
    LIMIT 1;

    IF result IS NOT NULL THEN
        RETURN result;
    END IF;

    -- Heuristic: if type contains LAND, LOT, or VACANT, treat as non-rentable
    IF UPPER(raw_type) ~ '(LAND|LOT|VACANT)' THEN
        RETURN 'land';
    END IF;

    RETURN 'other';
END;
$$;

-- Helper function: check if a property type is rentable
CREATE OR REPLACE FUNCTION is_rentable(raw_type TEXT)
RETURNS BOOLEAN
LANGUAGE sql
IMMUTABLE
STRICT
AS $$
    SELECT COALESCE(
        (SELECT r.is_rentable
         FROM property_type_rules r
         WHERE UPPER(r.raw_input_pattern) = UPPER(TRIM(raw_type))),
        -- Fallback: if contains LAND/LOT/VACANT, not rentable
        (UPPER(TRIM(raw_type)) !~ '(LAND|LOT|VACANT)')
    );
$$;
```

### 1i. Refactor Non-Rentable Checks in Python

**File:** `services/rent_estimator_v2.py`

Replace the hardcoded `NON_RENTABLE_TYPES` set + `is_non_rentable()` function

```python
def is_non_rentable(property_type: Optional[str], conn) -> bool:
    """Check property type against property_type_rules table."""
    if not property_type:
        return False

    try:
        cur = conn.cursor()
        cur.execute("""
            SELECT is_rentable FROM property_type_rules
            WHERE UPPER(raw_input_pattern) = UPPER(%s)
        """, (property_type.strip(),))
        row = cur.fetchone()
        cur.close()

        if row is not None:
            return not row[0]

        # Fallback heuristic if not found in rules table
        pt_upper = property_type.upper().strip()
        return any(x in pt_upper for x in ['LAND', 'LOT', 'VACANT'])
    except Exception:
        # DB unavailable — fall back to heuristic
        pt_upper = property_type.upper().strip()
        return any(x in pt_upper for x in ['LAND', 'LOT', 'VACANT'])
```

## Phase 2 — Rent Backfill & Auto-Update Pipeline

### 2a. Checkpointing Backfill Drain

**File:** `apps/worker/src/rent-estimator.ts`

**Problem:** `drain()` uses `ORDER BY id LIMIT $batch` — every restart starts from
the lowest pending ID. With 600k rows, this makes restart = full restart.

**Fix:** Add a Redis-based cursor so restarts resume where they left off:

```typescript
const CURSOR_KEY = 'rent_backfill_cursor';

async function drain(parentLog: WorkerLogger): Promise<number> {
  // Read last checkpoint from Redis, or start from 0
  let cursor: string | null = null;
  try {
    cursor = await redis.get(CURSOR_KEY);
  } catch { /* Redis down */ }

  const res = await pool.query<{ id: string }>(
    `SELECT id::text AS id
     FROM listings
     WHERE rent_calc_status = 'pending'
       ${cursor ? 'AND id > $2' : ''}
     ORDER BY id
     LIMIT $1`,
    cursor ? [env.RENT_BACKFILL_BATCH, cursor] : [env.RENT_BACKFILL_BATCH],
  );

  if (res.rowCount === 0) {
    // Nothing pending — reset cursor
    await redis.del(CURSOR_KEY).catch(() => {});
    return 0;
  }

  // Update cursor to the last dispatched ID
  const lastId = res.rows[res.rows.length - 1].id;
  await redis.set(CURSOR_KEY, lastId, 'EX', 3600).catch(() => {});

  for (const row of res.rows) runJob(row.id, parentLog);
  return res.rowCount ?? 0;
}
```

### 2c. Add Stale-Row Sweeper

```typescript
async function sweepStaleRows(): Promise<void> {
  const res = await pool.query<{ id: string }>(
    `SELECT id::text AS id
     FROM listings
     WHERE rent_calc_status = 'pending'
       AND updated_at < NOW() - INTERVAL '10 minutes'
       AND rent_retry_count < 3
```

## Phase 4 — Map Overhaul

### 4a. MVT Tile Migration — Full Sub-Plan

**Goal:** Replace the JSON-based `/api/properties/viewport` cluster API with
direct MVT (Mapbox Vector Tile) consumption from pg_tileserv. The viewport
endpoint serves JSON clusters at ~50–200KB per pan/zoom event. MVT tiles are
5–20KB per tile and cached by the browser.

**Decision: Direct tile serving via nginx subdomain (not proxied through Next.js).**
Proxying through Next.js adds 5–15ms latency per tile — at 50 tiles/page that is
250–750ms wasted. pg_tileserv is designed for direct, low-latency serving.

#### 4a.1 Fix the MVT Tile Function

**File:** `infrastructure/phase3_mvt_function.sql` → Move to `infrastructure/migrations/2026_06_16_mvt_tile_fix.sql`

**Problems identified in review:**
1. Does not include `estimated_rent` in tile payload — frontend can't display ratios
2. Does not filter non-rentable types — land lots appear as markers
3. `listing_status` parameter compared against `listing_type` column — confusing naming
4. File lives outside migration directory — can be missed

**Fix:**

```sql
-- Move to: infrastructure/migrations/2026_06_16_mvt_tile_fix.sql

-- Drop old function if it exists (from phase3_mvt_function.sql)
DROP FUNCTION IF EXISTS public.listings_mvt(integer, integer, integer, numeric, numeric, numeric, numeric, text);

CREATE OR REPLACE FUNCTION public.listings_mvt(
    z integer, x integer, y integer,
    min_price    numeric DEFAULT 0,
    max_price    numeric DEFAULT 999999999,
    min_beds     numeric DEFAULT 0,
    min_baths    numeric DEFAULT 0,
    listing_type_filter text DEFAULT 'for_sale',
    one_percent_only   boolean DEFAULT false,
    property_type_exclude text DEFAULT NULL
)
RETURNS bytea
LANGUAGE sql
IMMUTABLE STRICT PARALLEL SAFE
AS $$
    WITH mvt_geom AS (
        SELECT
            l.id,
            l.price,
            l.bedrooms,
            l.bathrooms,
            l.sqft,
            l.estimated_rent,
            l.address,
            l.property_type,
            l.listing_type,
            -- Ratio for the 1% rule — client uses this for color coding
            CASE WHEN l.price > 0 AND l.estimated_rent IS NOT NULL AND l.estimated_rent > 0
                 THEN l.estimated_rent / l.price
                 ELSE NULL
            END AS ratio,
            ST_AsMVTGeom(
                ST_Transform(l.geom, 3857),
                ST_TileEnvelope(z, x, y),
                4096, 256, true
            ) AS geom
        FROM listings l
        WHERE l.geom IS NOT NULL
          AND l.geom && ST_Transform(ST_TileEnvelope(z, x, y), 4326)
          AND l.listing_type = listing_type_filter
          AND l.price >= min_price
          AND l.price <= max_price
          AND l.bedrooms >= min_beds
          AND l.bathrooms >= min_baths
          -- Exclude non-rentable types
          AND (l.property_type IS NULL
               OR UPPER(l.property_type) NOT IN (
                 'LAND','LOT','LOTS','VACANT','VACANT_LAND','LOTS/LAND',
                 'FARM','MOBILE_LAND','OTHER','TIMBERLAND','AGRICULTURAL'))
          -- Optional: 1% rule only
          AND (NOT one_percent_only
               OR (l.price > 0
                   AND l.estimated_rent IS NOT NULL
                   AND l.estimated_rent > 0
                   AND (l.estimated_rent / l.price) >= 0.01))
    )
    SELECT ST_AsMVT(mvt_geom, 'properties', 4096, 'geom')
    FROM mvt_geom;
$$;

-- Grant execution so pg_tileserv can call it
GRANT EXECUTE ON FUNCTION public.listings_mvt(
    integer,integer,integer,numeric,numeric,numeric,numeric,text,boolean,text
) TO PUBLIC;
```
